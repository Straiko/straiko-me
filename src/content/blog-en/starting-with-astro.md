I wanted a portfolio that feels like a document: open it and it’s already there.

Most portfolio sites today behave like apps. That’s odd when the user came to read 1–2 screens of text.

Astro’s default is the right default: build pages into HTML and ship as little JavaScript as possible. Add interactivity as “islands” only where it pays off.

## The practical benefits

- Fast first render
- Simple routing (`src/pages`)
- Content as Markdown in the repo

## My rule

If a feature doesn’t make the site noticeably better for the reader, it doesn’t belong here.
