I learn fastest by building something slightly above my comfort zone.

If it’s too easy, you just reinforce what you already know. If it’s too hard, you drown in chaos. The sweet spot is in the middle.

## The loop

1. **Build**: implement a tiny feature until it works.
2. **Break**: stress it on purpose (bad input, empty data, flaky network).
3. **Fix**: make it robust, not “just patched”.
4. **Write**: capture the lesson while it’s fresh.

## What I write down

- what surprised me (that’s usually the real learning)
- where I wasted time
- which “small thing” broke everything
- 1–2 docs links I’ll come back to

Simple, but it compounds.
